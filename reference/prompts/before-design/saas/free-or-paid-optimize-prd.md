**元提示词 (Meta Prompt):**

```
## 元提示词：Freemium 功能点分析器

**目标：** 根据提供的产品需求文档 (PRD)，结合 Elena Verna 的增长策略原则，分析产品中的功能点，并明确建议哪些功能应作为免费增值 (Freemium) 的一部分提供，哪些应设为付费点。

**核心分析维度 (基于 Elena Verna 的观点):**

1.  **间接变现潜力 (Indirect Monetization Potential):**
    *   该功能是否有助于用户自发分享、邀请他人、或产生可吸引新用户的内容 (UGC)？
    *   该功能是否能增强产品的网络效应 (Network Effects)，即越多用户使用，产品对每个用户的价值越大？
    *   免费提供此功能是否能显著降低获客成本 (CAC)？

2.  **同质化程度 (Commoditization Level):**
    *   该功能是否已经是市场上的标配，或者竞争对手普遍免费提供？
    *   如果此功能收费，用户是否很容易找到免费的替代品？
    *   免费提供此功能是否是保持市场竞争力的基本要求？

3.  **“啊哈时刻” (Aha! Moment) 驱动:**
    *   该功能是否是用户首次体验产品核心价值的关键路径？
    *   免费提供此功能是否能让用户更快、更顺畅地理解“为什么这个产品对我有用”？
    *   缺少此功能是否会严重阻碍新用户激活？

4.  **习惯回路构建 (Habit Loop Formation):**
    *   该功能是否能鼓励用户高频使用？
    *   该功能是否包含通知、提醒、或促使用户定期返回产品的机制？
    *   免费提供此功能是否有助于将产品融入用户的日常工作流程或生活习惯？

5.  **增长模型摩擦 (Friction to Growth Model):**
    *   将此功能设为付费点，是否会严重阻碍用户增长、病毒传播或社区建设？
    *   此功能是否是用户进行下一步关键操作（例如邀请同事、分享成果）的前提？

6.  **核心价值主张 (Core Value Proposition) 与差异化:**
    *   该功能是否代表了产品的核心竞争力或独特的差异化优势？
    *   此功能是否是用户愿意为之付费的关键原因？
    *   如果免费提供此核心差异化功能，是否会削弱产品的整体付费意愿？

**输入：**

*   **PRD 文档内容 (Product Requirements Document):** [此处粘贴或引用PRD的关键部分，特别是功能列表、用户故事、目标用户描述、核心价值主张等]
*   **可选：产品所处行业和主要竞争对手信息：** [提供此信息有助于更精准地判断同质化程度]
*   **可选：产品当前阶段和增长目标：** [例如：早期寻求用户快速增长，还是成熟期寻求变现效率提升]

**输出要求：**

请针对 PRD 中列出的每一个主要功能点或功能模块，进行如下分析和建议：

1.  **功能点名称/描述：**
2.  **建议：** (免费 / 付费)
3.  **理由 (基于上述核心分析维度，请详细说明):**
    *   例如：
        *   “建议免费：此功能（如‘团队协作邀请’）能显著提升网络效应和用户间的病毒传播，有助于间接变现，降低获客成本。”
        *   “建议付费：此功能（如‘高级数据分析报告’）是产品的核心差异化价值，且针对的是有明确付费意愿的高价值用户群体，不属于基础的‘啊哈时刻’或习惯回路构建功能。”
        *   “建议免费：此功能（如‘基础任务管理’）在同类产品中已高度同质化，收费会失去竞争力。它同时也是用户体验核心价值、达成‘啊哈时刻’的基础。”
4.  **潜在的付费墙设计思考 (如果建议免费，但有升级可能):**
    *   例如：“基础版本免费，但限制项目数量/团队成员数量/存储空间，高级版提供无限额度。”
    *   例如：“核心查看功能免费，但编辑、导出等高级操作付费。”

**重要注意事项：**

*   请始终牢记 Elena Verna 的观点：“永远不要切换，总是叠加”。Freemium 的设计应服务于整体增长战略，并为付费转化铺路。
*   免费功能的目的是“喂养”增长引擎，而不是简单地“送东西”。
*   区分“对增长模型产生摩擦”和“提供核心付费价值”。有些功能虽然关键，但其价值足以支撑付费。

    *   文件共享 (每个任务50MB上限)
    *   项目甘特图视图
    *   自定义任务模板

    *   导出项目报告 (PDF, Excel)

**使用说明：**

将上述元提示词与具体的PRD文档内容结合，输入给大型语言模型 (LLM)，LLM 就能够根据 Elena Verna 的框架，对PRD中的功能点进行免费/付费的分析和建议。**你需要将方括号 `[]` 中的占位符替换为实际的 PRD 内容和相关信息。**

这个元提示词的关键在于它为 LLM 提供了一个清晰的分析框架和决策依据，使其不仅仅是随意猜测，而是基于一套行之有效的增长理论进行判断。
